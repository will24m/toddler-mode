# Chrome Web Store listing — submission crib sheet

Everything the CWS Developer Dashboard asks for, pre-written. Copy each section
into the matching dashboard field when submitting. Screenshots/promo images
still need to be captured by hand (1280×800 screenshots of the bubble on a
real page and of the options page work well).

## Single purpose (Privacy practices tab)

> Toddler Mode does one thing: when the user highlights text on a page and
> clicks the extension's icon, it shows a floating bubble with that text
> re-explained in very simple language, generated by an AI model (on-device
> Gemini Nano when available, otherwise a user-configured AI provider).

## Permission justifications

| Permission | Justification to paste |
|---|---|
| `storage` | Stores the user's own settings: AI provider choice, endpoint URL, and model name (sync storage), and the user's own API key (local storage only, never synced). Nothing else is stored. |
| Content script on `<all_urls>` | The extension's single purpose is to summarize text the user highlights on any page they are reading. The content script only watches for the user's own selection and draws the icon/bubble UI; it reads no page content beyond the text the user actively highlights. |
| `https://api.openai.com/*` | Default cloud endpoint: the highlighted text is sent to OpenAI's chat completions API with the user's own API key, only when the user clicks the extension's icon and only if on-device AI is unavailable. |
| `https://api.anthropic.com/*` | Same as above for the Anthropic API, the other built-in provider option. |
| Optional: `https://*/*`, `http://localhost/*`, `http://127.0.0.1/*` | Users may point the cloud fallback at their own OpenAI-compatible endpoint (e.g. a self-hosted LLM such as Ollama on localhost). The specific origin is requested at the moment the user saves that endpoint in the options page; nothing is requested for users of the built-in providers. |

## Data usage disclosures (Privacy practices tab)

- **Website content** — YES, collected: the text the user highlights is sent
  to the AI provider the user configured, solely to generate the summary.
  - Check: "Not being sold", "Not used for purposes unrelated to the item's
    core functionality", "Not used to determine creditworthiness".
- **Authentication information** — YES: the user's own API key, stored locally
  on the device and sent only to the user's chosen provider endpoint.
- Everything else (personally identifiable info, health, financial, location,
  web history, user activity, personal communications): **NOT collected**.
- Privacy policy URL: `https://github.com/will24m/toddler-mode/blob/main/PRIVACY.md`

## Remote code declaration

**No remote code.** All JavaScript ships in the package; the extension makes
plain `fetch` calls that return JSON/SSE data only, and never evaluates
fetched content as code.

## Description (store listing)

> Highlight anything → click the little bear 🧸 → get it explained like you're
> three years old.
>
> Toddler Mode turns dense text — legal boilerplate, academic papers, medical
> jargon — into two or three tiny sentences a small kid would understand. The
> summary streams into a friendly bubble right where you're reading, no new
> tabs.
>
> • Private by default: uses Chrome's built-in on-device AI (Gemini Nano) when
>   available — no account, no API key, nothing sent to any server.
> • Optional cloud fallback: plug in your own OpenAI, Anthropic, or
>   OpenAI-compatible endpoint and key for devices without on-device AI.
> • No tracking, no analytics, no data collection. Ever. Open source.

- Category: **Tools** (or **Accessibility**)
- Language: English

## Submission checklist

1. `npm run zip` → upload `.output/toddler-mode-<version>-chrome.zip`.
2. Paste the sections above into the dashboard.
3. Upload at least one 1280×800 screenshot.
4. Verify the developer contact email in the dashboard account tab.
5. Expect the "broad host permissions" review track because of the content
   script on `<all_urls>` — the single-purpose statement above addresses it;
   review may take a few extra days.
